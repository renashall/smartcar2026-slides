car_setup: the magic import

The car's hardware code lives in another folder — this line points Python at it.

Always first. import car_setup tells Python where the car's modules live.

After it, short imports like from led import Led just work — no long paths.

You never edit car_setup.py — you only import it.

lesson_1_components.py
"""Lesson 1: Components.

Run this on the Raspberry Pi (the car).
"""

import car_setup    # tells Python where the modules live
import time          # lets us pause with time.sleep()

The Pi is case-sensitive

Import names must match the real file names exactly — character for character.

Heads up
The file is servo.py (lowercase), so from servo import Servo works.
✓

from servo import Servo — matches the file.

✕

from Servo import Servo — works on Windows, fails on the Pi.

# each module controls one piece of hardware

Settings: a menu switch and the LED bitmask

DEMO_MODE picks what runs; the LED bitmask picks which of the 8 LEDs light up — one bit per LED.

lesson_1_components.py
# ---- settings you can change ----
DEMO_MODE = "all"   # leds / servo / distance / all

# pick which of the 8 LEDs to light up
LED_ALL = 0xFF   # all on
LED_OFF = 0x00   # all off
Value (hex) Binary Which LEDs
0xFF 1111 1111 all 8
0x00 0000 0000 none (off)
0x01 0000 0001 only LED 0 (first)
0x80 1000 0000 only LED 7 (last)
0x0F 0000 1111 first 4
0xAA 1010 1010 every other one
Hex

0x just means the number is written in hexadecimal — each hex digit = 4 bits, so 0xFF reads straight off as "8 ones = all LEDs." (0xFF = 255, 0x0F = 15, 0x80 = 128)

Name the parts, then setup() builds them

Create the hardware objects once, in one place.

We name the parts and set them to None — created later, together.

setup() fills them in and faces the head straight ahead.

global means "use the variables above — don't make new ones."

Channel is the string "0", not the number 0.

lesson_1_components.py
# ---- car parts (created in setup) ----
led = None
servo = None
ultrasonic = None

def setup():
    """Create the hardware objects once."""
    global led, servo, ultrasonic
    led = Led()                # the LED strip
    servo = Servo()            # the head servo
    ultrasonic = Ultrasonic()  # the distance sensor
    servo.setServoPwm("0", HEAD_CENTER)   # face forward

Demo 1: light the LEDs

Loop through red, green, blue — hold each a second — then turn off.

red green blue → off
lesson_1_components.py
def show_leds():
    """Red, then green, then blue, then off."""
    print("LED demo: red, green, blue, then off")
    for colour in (RED, GREEN, BLUE):
        red, green, blue = colour
        led.ledIndex(LED_ALL, red, green, blue)
        time.sleep(1)              # hold 1 second
    led.ledIndex(LED_ALL, 0, 0, 0)  # all off

Demo 2: sweep the head

Move left → center → right → center until the timer runs out, then re-center.

Sweep the positions over and over.

Stop after SERVO_DEMO_SECONDS, then face forward.

Check the clock each move so it stops on time.

lesson_1_components.py
def sweep_head():
    """Sweep, then re-center."""
    print("Servo demo: sweeping", SERVO_DEMO_SECONDS, "s")
    end_time = time.time() + SERVO_DEMO_SECONDS
    while time.time() < end_time:
        for angle in (HEAD_LEFT, HEAD_CENTER, HEAD_RIGHT, HEAD_CENTER):
            if time.time() >= end_time:
                break
            servo.setServoPwm("0", angle)
            time.sleep(0.6)
    servo.setServoPwm("0", HEAD_CENTER)

Demo 3: read the distance

Print the distance ahead every half-second until you press Ctrl+C.

read_distance() asks the sensor once and returns centimetres.

loop() prints it every 0.5s until Ctrl+C.

while True: means "repeat forever" — until we stop it.

Wave a hand in front and watch the number drop.

lesson_1_components.py
def read_distance():
    """Return the distance ahead, in cm."""
    return ultrasonic.get_distance()

def loop():
    """Print the distance until Ctrl+C."""
    print("Distance demo: press Ctrl+C to stop")
    while True:                    # repeat forever
        distance = read_distance()
        print("Distance ahead:", distance, "cm")
        time.sleep(0.5)

Clean up: destroy()

Always leave the car safe — LEDs off, head centered.

Leave the car safe and tidy: lights off, head centered.

A part may still be None if setup() failed — so check before using it.

Real robots should never be left in a weird state.

lesson_1_components.py
def destroy():
    """Leave the car safe when the program ends."""
    if led is not None:
        led.ledIndex(LED_ALL, 0, 0, 0)        # LEDs off
    if servo is not None:
        servo.setServoPwm("0", HEAD_CENTER)   # center

The main block ties it together

if __name__ == "__main__": runs only when you start this file directly.

try runs the chosen demos.

except catches Ctrl+C — no scary error.

finally always runs destroy() — even after an error.

lesson_1_components.py
if __name__ == "__main__":
    try:
        setup()
        if DEMO_MODE in ("leds", "all"):
            show_leds()
        if DEMO_MODE in ("servo", "all"):
            sweep_head()
        if DEMO_MODE in ("distance", "all"):
            loop()
        print("Component testing was completed.")
    except KeyboardInterrupt:
        pass                 # Ctrl+C: shut down quietly
    finally:
        destroy()            # always leave the car safe

Common issues

If something's off, it's almost always one of these.

Can't open /dev/mem · mmap() failed
You forgot sudo — the LED strip needs root.
No module named 'Servo'
Casing! It's from servo import Servo (lowercase file).
Head buzzes or jitters
Low battery, or it's fighting a wall — check power, give it room.
No distance / weird numbers
Sensor wires loose, or it's pointed at a soft/angled surface.
?
Stuck?

Check casing and sudo first — they cover most of today's snags. Then ask.
