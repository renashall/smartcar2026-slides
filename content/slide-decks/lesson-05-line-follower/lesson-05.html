Lesson 5 — Line follower · the clever bit

Three sensors → one number

Pack three yes/no answers into one number: Left×4 + Middle×2 + Right×1. Read it in binary as 0b L M R.

Pattern Binary Meaning
Middle only0b010centered → go straight
Left only0b100drifting right → turn left a bit
Left + Middle0b110turn left harder
Right only0b001drifting left → turn right a bit
Middle + Right0b011turn right harder
Why this is clever

Three on/off sensors give 8 possible patterns. One number captures all of them.

Each binary digit is one sensor: 0b L M R. This trick makes the steering code tiny.

Lesson 5 — Line follower · the file, chunk 4

setup(): the pins are inputs

Create the motor, then tell the Pi the IR pins are inputs — we READ them.

GPIO.BCM picks the pin-numbering scheme.

GPIO.IN = data flows from sensor → Pi.

setwarnings(False) hides harmless warnings.

lesson_5_line_follower.py
motor = None


def setup():
    """Make the motor; set sensor pins as inputs."""
    global motor
    motor = Motor()
    GPIO.setwarnings(False)   # hide harmless warnings
    GPIO.setmode(GPIO.BCM)     # BCM pin numbering
    GPIO.setup(IR_LEFT, GPIO.IN)    # IN = we READ these
    GPIO.setup(IR_MIDDLE, GPIO.IN)
    GPIO.setup(IR_RIGHT, GPIO.IN)

Lesson 5 — Line follower · the file, chunk 5

read_sensors(): combine the three

Read each pin (1 or 0), then pack them into one number — exactly the binary trick.

Each GPIO.input reads one sensor: 1 = line, 0 = floor.

Left×4 + Middle×2 + Right×1 builds the one number.

That number is the binary 0b L M R from slide 5.

lesson_5_line_follower.py
def read_sensors():
    """One number for the line's position."""
    left = GPIO.input(IR_LEFT)     # 1 = line
    middle = GPIO.input(IR_MIDDLE)
    right = GPIO.input(IR_RIGHT)
    return left * 4 + middle * 2 + right

Lesson 5 — Line follower · the file, chunk 6

choose_move(): one move per pattern

Match the sensor number to a steering move. The 0b… values read as L M R.

Read each 0b value out loud as L-M-R.

Anything else — all three, or none — → stop.

lesson_5_line_follower.py
def choose_move(sensors):
    """Pick wheel speeds for this pattern."""
    if sensors == 0b010:    # middle only -> straight
        return FORWARD
    if sensors == 0b100:    # left only -> left a bit
        return SOFT_LEFT
    if sensors == 0b110:    # left + middle -> harder
        return HARD_LEFT
    if sensors == 0b001:    # right only -> right a bit
        return SOFT_RIGHT
    if sensors == 0b011:    # middle + right -> harder
        return HARD_RIGHT
    return STOP            # all three, or none -> stop

loop(): sense and steer, fast

Read → decide → drive, over and over — the same Sense → Think → Act loop, just faster.

Read the sensors → decide the move → drive the motors.

The short sleep(0.02) (20 ms) keeps steering smooth.

Same loop as Lesson 4 — speed matters for smooth lines.

lesson_5_line_follower.py
def loop():
    """Read the sensors and drive, over and over."""
    while True:
        sensors = read_sensors()      # where is the line?
        move = choose_move(sensors)   # how to drive?
        motor.setMotorModel(move[0], move[1], move[2], move[3])
        time.sleep(0.02)             # keeps steering smooth

Clean up: destroy()

Stop the wheels, then release the pins with GPIO.cleanup().

setMotorModel(0, 0, 0, 0) stops the wheels first.

GPIO.cleanup() hands the pins back — it's new this lesson.

Releasing pins lets other programs use them next time.

lesson_5_line_follower.py
def destroy():
    """Stop the wheels and release the pins."""
    if motor is not None:
        motor.setMotorModel(0, 0, 0, 0)
    GPIO.cleanup()           # hand the pins back

The main block

The familiar safe wrapper: try runs the work, finally always cleans up.

try runs setup() then loop().

except KeyboardInterrupt catches Ctrl+C.

finally guarantees destroy() runs no matter what.

Fifth time seeing this block — you should predict it now.

lesson_5_line_follower.py
if __name__ == "__main__":
    try:
        setup()
        loop()
    except KeyboardInterrupt:
        pass
    finally:
        destroy()

Common issues

If the following looks off, it's almost always one of these.

Ignores the line
Check the IR sensors face the floor; is the line dark enough?
Steers the wrong way
Left/Right sensors may be swapped — check pins 14 / 15 / 23.
Stops constantly
Reading "all" or "none" — adjust sensor height or track contrast.
"Pin already in use"
Harmless — GPIO.setwarnings(False) hides it.
Check this first

Sensor height above the floor matters a lot — a few millimeters can fix erratic reads.
