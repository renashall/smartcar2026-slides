What is an ADC?

It turns a smooth analog voltage into a number the computer can read.

Sensors like light and battery give a smooth analog voltage.

But computers only understand digital numbers.

The ADC — Analog-to-Digital Converter — translates voltage into a number.

We read it with adc.recvADC(channel), which returns volts.

Daemons and a stop flag

Two small ideas keep our background thread tidy — and let us shut it down cleanly.

Daemon thread
Quits with the program

A daemon thread is allowed to stop automatically when the main program ends, so it never holds the program open.

Event
A simple stop switch

An Event is an on/off flag. We flip it to tell the thread "time to stop" — a clean shutdown.

Why it matters

A clean shutdown means we never leave a thread running or the ADC half-open.

setup(): build and start the thread

Three steps for a thread: make it, set it as a daemon, then start it.

Create the sensors, then make and start the background thread.

target=monitor_battery = "run that function in the thread."

daemon = True = quit with the program. .start() launches it.

lesson_3_adc_multithreading.py
def setup():
    """Create sensors and start the battery thread."""
    global adc, buzzer, battery_thread
    adc = Adc()
    buzzer = Buzzer()

    battery_thread = Thread(target=monitor_battery)
    battery_thread.daemon = True   # stop with the program
    battery_thread.start()         # begin running now

Read and scale the battery

Read channel 2, multiply by 3, then turn the voltage into a rough percent.

recvADC(BATTERY) gives volts; × 3 is the real pack voltage.

int(...) drops the decimals to a whole percent.

Maps 7.0 V → 0% and 8.4 V → 100%.

lesson_3_adc_multithreading.py
def read_battery_voltage():
    """Scale the ADC reading to the real pack voltage."""
    return adc.recvADC(BATTERY) * 3   # ADC sees 1/3


def battery_percent(voltage):
    """Turn a voltage into a rough 0–100% estimate."""
    return int((voltage - 7) / 1.40 * 100)   # 7.0=0% 8.4=100%

beep(): a reusable helper

Beep on and off a chosen number of times — reusing the buzzer strings from Lesson 2.

buzzer.run("1") turns it on, "0" turns it off — same as Lesson 2.

_ is a throwaway name — we just want to repeat.

Pass 2 for low, 4 for critical.

lesson_3_adc_multithreading.py
def beep(times):
    """Beep on and off a chosen number of times."""
    for _ in range(times):   # "_" = a name we don't use
        buzzer.run("1")      # on
        time.sleep(0.1)
        buzzer.run("0")      # off
        time.sleep(0.1)

monitor_battery(): the background loop

This is the function the second thread runs — it loops until told to stop.

while not stop_event.is_set() = loop until told to quit.

Beeps more the lower it gets: critical → 4, low → 2.

Waits 3 seconds between checks.

lesson_3_adc_multithreading.py
def monitor_battery():
    """Runs in the background thread."""
    while not stop_event.is_set():        # loop until stopped
        voltage = read_battery_voltage()
        print("Battery:", round(voltage, 2), "V")
        if voltage <= CRITICAL_BATTERY:
            beep(4)
        elif voltage <= LOW_BATTERY:
            beep(2)
        time.sleep(3)                     # check again in 3 s

loop(): the main thread

While the battery thread runs quietly, the main thread keeps printing the light sensors.

Reads both light channels every second.

round(value, 2) keeps the printout readable.

Now there are two while loops alive at once — multithreading in action.

lesson_3_adc_multithreading.py
def loop():
    """The MAIN thread: print the two light readings."""
    while True:
        left = adc.recvADC(LEFT_LIGHT)
        right = adc.recvADC(RIGHT_LIGHT)
        print("Light  left:", round(left, 2),
              "  right:", round(right, 2))
        time.sleep(1)

Clean up: destroy()

Flip the stop flag, wait for the thread to finish, then close the ADC.

stop_event.set() tells monitor_battery() to finish.

join(timeout=2) = wait up to 2 s so we never hang forever.

i2cClose() closes the connection to the ADC chip.

lesson_3_adc_multithreading.py
def destroy():
    """Stop the thread and close the sensors cleanly."""
    stop_event.set()             # tell the thread to finish
    if battery_thread is not None:
        battery_thread.join(timeout=2)   # wait (max 2 s)
    if buzzer is not None:
        buzzer.run("0")          # make sure it's off
    if adc is not None:
        adc.i2cClose()           # close the ADC connection

The main block

The same trusty pattern: try runs the work, finally always cleans up.

try runs setup() then loop().

except KeyboardInterrupt catches Ctrl+C.

finally guarantees destroy() runs no matter what.

Third lesson in a row with this block — you should recognise it instantly.

lesson_3_adc_multithreading.py
if __name__ == "__main__":
    try:
        setup()
        loop()
    except KeyboardInterrupt:
        pass
    finally:
        destroy()

Common issues

If something's off, it's almost always one of these.

Weird battery voltage
Remember the × 3, and check the ADC wiring.
Won't quit on Ctrl+C
That's the join(timeout=2) wait — give it a second.
i2c / OSError
I²C not enabled, or the ADC address / wiring is off.
Light values don't change
Sensor covered already, or wired to the wrong channel.
Check I²C

Most ADC errors come from I²C being disabled. Enable it in raspi-config and reseat the ADC.
