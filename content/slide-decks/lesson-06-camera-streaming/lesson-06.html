Server · chunk 3 / 7

setup(): open the door, start the camera

Make a socket, claim port 8000, listen — then start the camera and let it warm up.

Make a door, claim a number, listen.

SO_REUSEADDR lets us restart cleanly.

Then camera.start() and a short warm-up.

lesson_6_pi_camera_stream_server.py
server_socket = connection = camera = None


def setup():
    """Open the doorway and start the camera."""
    global server_socket, camera
    server_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    server_socket.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
    server_socket.bind((HOST, VIDEO_PORT))   # claim the port
    server_socket.listen(1)                  # wait for 1 connection

    camera = Picamera2()
    camera.configure(camera.create_video_configuration(
        main={"size": (FRAME_WIDTH, FRAME_HEIGHT)}))
    camera.start()
    time.sleep(WARMUP_SECONDS)               # let it settle

accept_client(): wait for the computer

def accept_client():
    """Wait for the computer, then open the pipe."""
    global connection
    client_socket, address = server_socket.accept()
    connection = client_socket.makefile("wb")
    print("Computer connected from", address[0])

Server · chunk 5 / 7

loop(): capture & send each frame

Take a JPEG into memory, then send size first, picture second — the protocol from slide 5.

Capture one JPEG into an in-memory file.

struct.pack("<L", length) makes the 4 size-bytes.

Then write the picture bytes after the size.

lesson_6_pi_camera_stream_server.py
def loop():
    """Capture forever; send each picture down the pipe."""
    stream = io.BytesIO()                # in-memory file
    while True:
        stream.seek(0); stream.truncate()   # empty it
        camera.capture_file(stream, format="jpeg")  # one JPEG

        length = stream.tell()             # how many bytes
        connection.write(struct.pack("<L", length))  # size
        connection.flush()                 # push it out now

        stream.seek(0)                    # back to the start
        connection.write(stream.read())    # the picture bytes

Server · chunk 6 / 7

destroy(): send "done" & close

Send a size of 0 as the agreed stop signal, then close everything tidily.

A size of 0 tells the viewer to stop.

The computer may already be gone — wrap it in try.

Close the camera and the socket cleanly.

lesson_6_pi_camera_stream_server.py
def destroy():
    """Tell the computer we're done and close up."""
    if connection is not None:
        try:
            connection.write(struct.pack("<L", 0))  # 0 = stop
        except OSError:
            pass                         # computer may be gone
        connection.close()
    if camera is not None:
        camera.stop(); camera.close()
    if server_socket is not None:
        server_socket.close()

Viewer · chunk 2 / 5

setup(): read the IP & connect

You type the Pi's IP on the command line; sys.argv[1] reads it. Port must match the server.

Connect to port 8000 — same as the server.

makefile("rb") = read-bytes like a file.

Use 127.0.0.1 if the viewer runs on the Pi.

lesson_6_client_video_receiver.py
VIDEO_PORT = 8000                 # match the Pi server
WINDOW_NAME = "Lesson 6 - Pi Camera"
video_socket = video_file = None


def setup():
    """Read the Pi's address and connect."""
    global video_socket, video_file
    if len(sys.argv) < 2:
        raise SystemExit("Pass the Pi's IP, e.g. python this.py 192.168.1.50")
    pi_ip = sys.argv[1]

    video_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    video_socket.connect((pi_ip, VIDEO_PORT))   # dial the Pi
    video_file = video_socket.makefile("rb")    # read bytes

Viewer · chunk 3 / 5

read_frame(): size, then picture

This mirrors the server exactly: server packs the size, viewer unpacks it.

Read the 4 size-bytes, turn them into a number.

If the size is 0 → the stream ended.

Else read that many bytes and decode the JPEG.

lesson_6_client_video_receiver.py
def read_frame():
    """Read one picture, or None when the stream ends."""
    header = video_file.read(4)            # the 4 size-bytes
    if len(header) != 4:
        return None
    length = struct.unpack("<L", header)[0]  # bytes -> number
    if length == 0:
        return None                       # the Pi's "done" signal

    jpeg = video_file.read(length)       # exactly that many bytes
    if len(jpeg) != length:
        return None
    return cv2.imdecode(np.frombuffer(jpeg, np.uint8), cv2.IMREAD_COLOR)

Viewer · chunk 4 / 5

loop(): show the video

Read a frame, show it, repeat. Press q to quit.

read_frame() returns None → the stream ended.

imshow draws the picture in the window.

waitKey(1) refreshes it and checks for q.

lesson_6_client_video_receiver.py
def loop():
    """Show pictures until the video ends or you quit."""
    while True:
        frame = read_frame()
        if frame is None:
            break                      # no more pictures
        cv2.imshow(WINDOW_NAME, frame)  # draw it
        if cv2.waitKey(1) & 0xFF == ord("q"):
            break

Common issues

If the stream looks off, it's almost always one of these.

Connection refused
Start the server first, and check the IP / port (8000).
Black or frozen window
Camera not ready — check the camera cable and picamera2.
Laggy video
Lower the resolution / frame rate, or get closer to the router.
ModuleNotFoundError: cv2
Install opencv-python and numpy on the viewer.
Check this first

90% of issues are viewer started before server or a wrong IP. Check those two first.
