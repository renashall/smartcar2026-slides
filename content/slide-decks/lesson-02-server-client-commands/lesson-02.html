Big idea: a client and a server

An app sends orders; the car carries them out. Today we play both roles in one file.

Client — an app on your laptop or phone — sends an order like "drive forward".

Server — the car — does it with the motors.

Think of it like a restaurant: the client orders, the server (kitchen) cooks.

Today both roles live in one file: we build the order, then run it on the car.

Commands are just text

The client and server talk with short strings, glued together with #.

Nothing magic travels over the wire — it's only a string.

A command name comes first, then its values.

The pieces are joined with a # between each one.

We'll write a tiny helper that builds these so we can see the message.

CMD_MOTOR#1500#1500#1500#1500
set the four wheels to 1500
CMD_BUZZER#1
buzzer on
CMD_LED#255#0#0#255
set the LEDs to a color

Settings: the speed and the moves

Each move is four wheel values in LF / LB / RF / RB order.

Forward = all positive · Backward = all negative.

Name the parts, then setup()

Same pattern as Lesson 1 — set names to None, then build them in one place.

motor = None
buzzer = None
led = None
cmd = None

def setup():
    """Create the parts and command helper."""

    motor = Motor()
    buzzer = Buzzer()
    led = Led()
    cmd = COMMAND()       # names like cmd.CMD_MOTOR

Build the command string yourself

This is the heart of the lesson: glue a name and its values with #.

["CMD_MOTOR", 1500, 1500]
↓ join with "#"
CMD_MOTOR#1500#1500

Start with the name, add each value as text.

"#".join(parts) puts a # between every piece.

lesson_2_server_client_commands.py
def format_command(name, values):
    """Build CMD_MOTOR#1500#1500."""
    parts = [name]              # start with the name
    for value in values:       # add each value as text
        parts.append(str(value))
    return "#".join(parts)     # glue with "#"

drive(): print, then move

Show the command first — then actually turn the wheels.

First print the command so students see it.

Then setMotorModel(lf, lb, rf, rb) drives the four wheels.

Connect the printed line to the wheels turning — message → action.

def drive(move, seconds):
    """Run one move, printing its command."""
    print("Sending:", format_command(cmd.CMD_MOTOR, move))
    motor.setMotorModel(move[0], move[1], move[2], move[3])
    time.sleep(seconds)

test_buzzer(): beep on, then off

The buzzer uses strings — "1" for on, "0" for off.

Heads up
Pass the string "1", not the number 1.
✓

buzzer.run("1") — correct.

✕

buzzer.run(1) — a common silent bug.

lesson_2_server_client_commands.py
def test_buzzer():
    """Beep on, then off."""
    print("Sending:", format_command(cmd.CMD_BUZZER, ["1"]))
    buzzer.run("1")       # "1" = on, "0" = off (strings!)
    time.sleep(1)
    buzzer.run("0")

loop(): the whole routine

Every move in order, then beep, then flash — once.

Run each move, then the buzzer and LED tests.

We drive(STOP, 1) before the tests — no rolling during the beep.

Each helper still prints its own Sending: line.

def loop():
    """Drive, beep, then flash — once."""
    drive(FORWARD, 1)
    drive(BACKWARD, 1)
    drive(TURN_LEFT, 1)
    drive(TURN_RIGHT, 1)
    drive(STOP, 1)        # stop before the tests
    test_buzzer()
    test_leds()

Clean up: destroy()

Always finish stopped and quiet — motors off, buzzer off, LEDs off.

Stopping the motors here is a safety must.

A crashed program should never leave the wheels spinning.

Guard each part with is not None in case setup failed.

lesson_2_server_client_commands.py
def destroy():
    """Leave the car stopped and quiet."""
    if motor is not None:
        motor.setMotorModel(0, 0, 0, 0)   # stop wheels
    if buzzer is not None:
        buzzer.run("0")                   # silence
    if led is not None:
        led.ledIndex(LED_ALL, 0, 0, 0)    # LEDs off

The main block ties it together

Same safe pattern as Lesson 1 — try, except, finally.

try runs setup() then loop().

except catches Ctrl+C — no scary error.

finally always runs destroy() — even after a crash.

lesson_2_server_client_commands.py
if __name__ == "__main__":
    try:
        setup()
        loop()
    except KeyboardInterrupt:
        pass                  # Ctrl+C: stop quietly
    finally:
        destroy()             # always leave the car safe

Common issues

If something's off, it's almost always one of these.

Car drives off the table
Lift the wheels first; lower DRIVE_SPEED.
Wheels weak or uneven
Usually a low battery, or a loose motor wire.
Buzzer is silent
You passed 1 instead of the string "1".
ModuleNotFoundError
Casing, or car_setup not imported first.
Check the battery

Weak or uneven driving is almost always low power. Keep charged spares ready.
